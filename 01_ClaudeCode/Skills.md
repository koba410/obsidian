# Claude Code の Skills について

## 1. Skills（スキル）とは何か？

簡単に言うと、**よく使う複雑な指示を、1つのコマンド（ショートカット）として登録したもの**です。通常なら長いプロンプトを打つ必要がある作業を、独自のスキルとして定義しておくことで、次からは一言で実行できるようになります。

### 基本的な登録方法

ターミナルで以下のように指示するだけで、Claudeが新しいスキルを学習します。

> 「新しいスキル create-api-resource を覚えて。指示内容は：引数にモデル名を受け取ったら、1.Migration、2.Controller (API用)、3.Resourceクラス、4.Pestテストを、弊社のコーディング規約（DOCS.md参照）に沿って一括生成すること。」

## 2. Laravel開発で「神スキル」になる4つの具体例

バックエンド業務で特におすすめのスキルセットです。

### ① db-refactor（DB設計の自動反映）

マイグレーションファイルを修正した際、関連する全箇所を直すスキルです。

- **内容:** マイグレーションの変更を検知し、Modelの`$fillable`、Factory、Resource、バリデーションルールを一括更新する。
- **メリット:** 「カラム追加し忘れた！」という時の修正漏れがゼロになります。

### ② secure-scan（セキュリティ特化レビュー）

1年目だと見落としがちな脆弱性を、プロの視点でチェックさせるスキルです。

- **内容:** PR（プルリクエスト）作成前に、SQLインジェクション、N+1問題、認可（Policy）の漏れ、機密情報のログ出力を重点的にスキャンする。
- **メリット:** シニアエンジニアからの指摘（マサカリ）を事前に防げます。

### ③ make-pest-scenario（テストシナリオ作成）

テストを書く手間を極限まで減らすスキルです。

- **内容:** Controllerのメソッドを読み取り、正常系だけでなく、403(認可エラー)や422(バリデーションエラー)のテストケースを自動生成する。
- **メリット:** テストのカバレッジが爆上がりします。

### ④ doc-sync（ドキュメント同期）

コードと仕様書の乖離（かいり）を防ぐスキルです。

- **内容:** コードの変更内容から、openapi.yaml（Swagger）や社内Wiki用のMarkdownを自動更新する。
- **メリット:** 「仕様書が古い」と怒られることがなくなります。

---

## 3. Skillsを管理・共有するテクニック

2026年のアップデートで、スキルは個人用だけでなく、プロジェクト共有が可能になりました。

- **`.claude/skills/` ディレクトリ:** ここにスキルの定義（JSONやMarkdown形式）を保存してGit管理することで、チームメンバー全員が同じ自作コマンドを使えるようになります。
- **スキルの呼び出し:** `claude "run skill secure-scan"` のように呼び出せます（最近はもっと短く `claude #secure-scan` のようなエイリアスも流行っています）。

---

## まとめ

繰り返し作業が発生する部分にskillsを実装し、効果が見込めたらチームメンバー全員が実行できるようにgitで共有すれば、便利なノウハウが次々と溜まっていくのではないかと思います。
