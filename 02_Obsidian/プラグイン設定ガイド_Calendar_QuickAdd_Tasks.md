# Calendar / QuickAdd / Tasks 設定ガイド（日本語）

各プラグインの設定画面に表示される項目を、**英語名 → 日本語の説明**でまとめています。設定画面が英語でもこの一覧を見ながら設定できます。

---

## Calendar（カレンダー）

※ liamcain 氏の「Calendar」プラグインです。デイリーノートの日付をクリックして開くカレンダー表示を追加します。

| 設定項目（英語） | 日本語の説明 |
|------------------|--------------|
| **Start week on** | **週の開始曜日** … カレンダーで週の最初を「日曜」にするか「月曜」にするか。`Locale` は Obsidian の言語設定に従う（日本語なら日曜始まりが多い）。 |
| **Words per Dot** | **1ドットあたりの文字数** … 日付の下のドットは「その日のノートの文字量」を表す。250 なら 250 語で 1 ドット。`0` にするとドット表示をオフ。 |
| **Confirm before creating new note** | **新規ノート作成前に確認する** … 日付をクリックしてデイリーノートを作る際、確認ダイアログを出すか。オフにすると即作成。 |
| **Show Week Number** | **週番号を表示する** … カレンダーに週番号の列を出し、クリックで週次ノートを開けるようにする。週次ノートを使う場合は ON にすると設定が表示される。 |

**週次ノート（Weekly notes）について**  
週次ノート機能は別プラグイン「Periodic Notes」に移行推奨です。Calendar 設定で「Show Week Number」を ON にすると、従来どおり週次ノートの **Folder**（保存フォルダ）・**Template**（テンプレート）・**Format**（ファイル名の日付形式）が設定できます。

## QuickAdd（クイックアド）

テンプレート挿入・キャプチャ・マクロを「選択肢（Choice）」として登録し、コマンドパレットやホットキーで実行するプラグインです。

### 設定画面の構成

- **Manage Choices** … 選択肢の一覧。ここに「Template」「Capture」「Macro」「Multi」を追加する。
- **Show QuickAdd in ribbon** … リボン（左サイドバー）に QuickAdd アイコンを表示するか。
- **Run QuickAdd on load** … 起動時に QuickAdd の選択肢メニューを自動で開くか（通常はオフ）。

### 選択肢の種類（Choice type）

| 種類（英語） | 日本語の説明 |
|--------------|--------------|
| **Template** | **テンプレート** … 指定したテンプレートを現在のノートに挿入、または新規ノートとして作成する。 |
| **Capture** | **キャプチャ** … 入力したテキストを指定フォルダ・ファイルに追記する（メモの即記録向き）。 |
| **Macro** | **マクロ** … 複数の処理（コマンド・テンプレート・スクリプトなど）を順番に実行する。 |
| **Multi** | **マルチ** … 他の選択肢をフォルダのようにまとめ、一覧から選べるようにする。 |

### 選択肢を追加したあとの主な項目

| 項目（英語） | 日本語の説明 |
|--------------|--------------|
| **Name** | **名前** … コマンドパレットなどに表示される名前。 |
| **Template path** | **テンプレートのパス** … 使うテンプレートファイル（例: `03_Template/Memo_Template`）。 |
| **File name format** | **ファイル名の形式** … 新規ノートで使うファイル名。`{{DATE:YYYY-MM-DD}}` で日付など。 |
| **Folder** | **フォルダ** … 新規ノートやキャプチャの保存先フォルダ。 |
| **Open in new pane** | **新しいペインで開く** … 新規ノートを別ペインで開くか。 |
| **Hotkey** | **ホットキー** … Obsidian の「ホットキー」設定で、この選択肢にキーを割り当てる。 |

### フォーマットで使える主な変数（テンプレート内）

| 記法 | 意味 |
|------|------|
| `{{DATE}}` / `{{DATE:YYYY-MM-DD}}` | 今日の日付（形式指定可） |
| `{{VALUE}}` / `{{NAME:プロンプト文}}` | 入力プロンプトでユーザーが入力した値 |
| `{{TIME}}` | 現在時刻 |
| `{{LINKCURRENT}}` | 現在開いているノートへのリンク |

## Tasks（タスク）

チェックボックス（`- [ ]` / `- [x]`）をタスクとして集約し、期限・繰り返し・クエリで表示するプラグインです。

### 主な設定カテゴリと項目

#### タスクの形式・フィルタ

| 設定項目（英語） | 日本語の説明 |
|------------------|--------------|
| **Global filter** | **グローバルフィルタ** … 「タスク」として認識する条件。例: `#task` と入れると、このタグが付いた行だけが Tasks の対象になる。空欄ならすべての `- [ ]` をタスクとして扱う。 |
| **Set created date on every added task** | **追加したタスクに作成日を付ける** … 「Create or edit task」でタスクを作成したとき、作成日を自動で付けるか。 |
| **Set done date on every completed task** | **完了したタスクに完了日を付ける** … チェックを入れたときに `✅ 2025-02-18` のように完了日を自動追記するか。**おすすめ: ON**。 |

#### 日付まわり

| 設定項目（英語） | 日本語の説明 |
|------------------|--------------|
| **Use filename as default date** | **ファイル名をデフォルトの日付にする** … デイリーノート（例: `2026-02-18.md`）内のタスクを、そのファイル名の日付のタスクとして扱う。デイリーノート運用なら **ON** が便利。 |
| **Date format** | **日付形式** … タスクの日付表示形式（例: `YYYY-MM-DD`）。 |

#### ステータス（状態）

| 設定項目（英語） | 日本語の説明 |
|------------------|--------------|
| **Status settings** | **ステータス設定** … 未完了 `[ ]`・完了 `[x]` 以外に、独自の記号（例: `[/]` 進行中）を追加できる。各ステータスに「次の状態」「TODO / DONE / IN_PROGRESS / CANCELLED」を指定可能。 |

#### 繰り返しタスク

| 設定項目（英語） | 日本語の説明 |
|------------------|--------------|
| **Recurring tasks** 関連 | **繰り返しタスク** … `🔁 毎週` のような繰り返しルールを解釈し、完了時に次回分を生成する。設定で挙動を調整可能。 |

#### その他

| 設定項目（英語） | 日本語の説明 |
|------------------|--------------|
| **Auto-suggest** | **自動提案** … タスク編集時に日付や繰り返しの候補を出すか。 |
| **Create or edit Task modal** | **タスク作成・編集モーダル** … 「Tasks: Create or edit task」で開くダイアログの項目や見た目をカスタマイズする。 |

※ 変更後は **Obsidian の再起動**や**Vault の再オープン**が必要な項目があります。

---

## このノートの使い方

- 設定 → コミュニティプラグイン → Calendar / QuickAdd / Tasks の「歯車」で設定を開く。
- 画面に表示されている英語の項目名を、このガイドの表で探し、日本語の説明を確認しながら設定する。
