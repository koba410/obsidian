# Obsidian 運用ルール

このVaultでノートを書く際に守るルールと、推奨する慣習です。
詳細なベストプラクティスは [[Obsidianベストプラクティス]] を参照。

---

## フォルダ構成

```
Vault/
├── 00_Daily/        # デイリーノート（YYYY-MM-DD.md）
├── 01_ClaudeCode/   # ClaudeCode 関連ノート
├── 02_Obsidian/     # Obsidian 設定・使い方
├── 03_Template/     # テンプレート置き場
├── 04_memo/         # クイックメモ・未分類
├── 05_book/         # 書籍メモ
└── 99_Archive/      # 完了・非アクティブなノート
```

**フォルダは "入れ物"、タグは "属性"** で管理する。

---

## 必須ルール

### 1. frontmatter を書く

ノート作成時は YAML frontmatter を先頭に入れる：

```yaml
---
date: 2026-02-18
tags:
  - type/memo
  - status/inbox
---
```

### 2. タグは階層タグで統一

```
type/daily     type/memo     type/project     type/book
status/inbox   status/active  status/done     status/archive
area/work      area/learning  area/personal
tool/obsidian  tool/claude    tool/git
```

1ノートに 3〜5 タグまで。少なく深く。

### 3. フォルダごとに 001_Index.md を置く

- **役割**: そのフォルダの目次として、配下のノートへのリンクを一覧する
- ノートを追加したら Index にもリンクを追記する

### 4. テンプレートを使う

新規ノートはテンプレートから作成する：

| 用途 | テンプレート |
|------|-------------|
| デイリーノート | `Daily_Template.md` |
| メモ・気づき | `Memo_Template.md` |
| プロジェクト | `Project_Template.md` |
| 書籍メモ | `Reading_Template.md` |
| タスク管理 | `Task_Template.md` |

---

## 推奨ルール

### 5. ファイル名の付け方

| タイプ | 命名例 |
|--------|--------|
| デイリーノート | `2026-02-18.md` |
| プロジェクト | `プロジェクト名.md` |
| メモ | `わかりやすい日本語タイトル.md` |

- 日本語OK。スペースより `_` を推奨
- 避ける文字: `#`, `[`, `]`, `^`, `|`, `\`

### 6. リンクでノートをつなぐ

- `[[ノート名]]` でリンクを積極的に使う
- 書きながら `[[` を打つ習慣をつける
- 孤立したノートを減らす（グラフビューで確認）

### 7. フォルダ階層は浅く

- 最大 2〜3 階層まで
- 深い分類はタグや MOC で代替する

### 8. アーカイブを活用する

- 完了したプロジェクト・古いノートは `99_Archive/` へ移動
- 削除より退避を優先する

### 9. 週次レビュー（推奨）

週に1度、15〜30分で：
1. デイリーノートを振り返り、重要メモを専用ノートへ昇格
2. 完了プロジェクトを `99_Archive/` へ移動
3. Inbox（未整理ノート）をゼロにする
4. 001_Index を更新する
