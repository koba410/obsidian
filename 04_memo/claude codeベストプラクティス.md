## CLAUDE.md を起点にした開発のベストプラクティス

### 1. まず `CLAUDE.md` を育てることを最優先にする

チーム全員が `CLAUDE.md` に気づいたことを書き足していくことが大切です。コーディング規約、よく使うコマンド、プロジェクト固有の注意点など、Claude に「知っておいてほしいこと」を何でも書く場所として運用します。最初は雑然としていても問題ありません。

### 2. 肥大化したら Claude 自身に分割を提案させる

`CLAUDE.md` が100行を超えてきたら、Claude に以下のようなプロンプトを投げるのが効果的です。

```
CLAUDE.md の内容を読んで、rules / skills / agents などのファイルに
分割する構成を提案してください。
```

Claude が `CLAUDE.md`（メインの設定）と、`rules/`・`skills/`・`agents/` などのサブファイルへの整理案を出してくれるので、それをチームでレビューして採用します。

### 3. Git で共有・バージョン管理する

`CLAUDE.md` はソースコードと同様にリポジトリに含めてバージョン管理します。変更は PR・レビューを通じて行うことで、チーム全員が内容を把握できます。

### 4. 「Claude が失敗したとき」をフィードバックの起点にする

Claude が期待どおりに動かなかったときに「なぜ失敗したか」を分析し、そのコンテキストや制約を `CLAUDE.md` に追記するサイクルを回すと、ファイルが自然と洗練されていきます。

### 5. 推奨ファイル構成のイメージ

```
CLAUDE.md          # メインファイル（〜100行）。ここを起点にする
.claude/
  rules/           # コーディング規約・禁止事項など
  skills/          # よく使うタスクのテンプレ・手順
  agents/          # サブエージェントの役割定義
```

---

**要点をまとめると**、最初から完璧な構造を目指すのではなく、「まず `CLAUDE.md` に書く → 肥大化したら Claude に整理させる → チームでレビューして採用する」というサイクルを回すのが、現実的かつ効果的な進め方です。
